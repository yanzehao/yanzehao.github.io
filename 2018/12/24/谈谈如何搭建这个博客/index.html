    前言

    启蒙

    作为95后,对于博客的印象比较模糊,我的互联网启蒙时期大约在2008年,那时候流行QQ空间,同学们之间互踩空间成为时尚.当时特别留意谁看过我,默默期待着这份名单中能出现她的名字.

    QQ空间成为了我在互联网上的第一块小小的阵地,美化QQ空间成为了我当时最大的乐趣,这当然是在不开黄钻的前提下进行的,极客精神在此时慢慢萌芽.

    动机

    graph TD;
A(博客:内容题材不限); B(QQ空间:信息冗余); C(微博:140字以内); D(朋友圈/说说:无图无JB)
    A-->B;
    B-->C;
    C-->D;

    互联网的高速发展,人们沟通方式越来越便利的同时,所承载的信息量越来越匮乏,一张图片配上一行文字足以记录一天的心情,可生活总不能如此琐碎,偶尔需要好好整理一番.博客这个’古老’的工具,似乎挺适合充当日记的职责:记录一些不好意思向所有人分享的想法.

    思路

    设计风格

    起初想自己从无到有自己写一个博客系统,但发现涉及到的知识量远超于我目前的水平,就UI设计这一块,对于我来说是完全空白的,但我还是有一些自己的偏好和思考.

    • 微软UWP应用

      在4年前,我使用过一款诺基亚的智能机,搭载的是Window Phone系统,其中Modern UI别具一格,随后Microsoft 推出window10系统,我的诺基亚升级后,原生的UWP应用体验令人拍手称赞,使用UWP的过程中其UI给我留下深刻印象

      • 跨平台
      • 简洁高效

    • Material Design

      只要稍微了解一下Google,无不被这家公司的伟大所折服,没错,我就是Google的脑残粉,其中Material Design是Google创造的一种独一无二的底层系统，在这个系统的基础之上，构建跨平台和超越设备尺寸的统一体验。遵循基本的移动设计定则，同时支持触摸、语音、鼠标、键盘等输入方式。

      • 跨平台和超越设备尺寸的统一体验

      • 鲜明,形象,简洁的动画彰显年轻

    • 我的需求

      • 设计简约,注重阅读体验.
      • 自适应和响应式设计,PC端和移动端均能良好适配.
      • 充分利用现有开发工具,降低博客搭建难度,保证写作效率.
      • 对Google存在一定的信仰加成

      出于以上因素的考量,倾向于使用Material Design风格,其实我想说:Google大法好!

    写作工具

    • markdown

      • Markdown是一种轻量级标记语言，它以纯文本形式(易读、易写、易更改)编写文档，并最终以HTML格式发布。
      • Markdown也可以理解为将以MARKDOWN语法编写的语言转换成HTML内容的工具。
      • 专注你的文字内容而不是排版样式，安心写作
      • 轻松的导出 HTML、PDF 和本身的 .md 文件
      • 纯文本内容，兼容所有的文本编辑器与字处理软件
      • 可读、直观、学习成本低

    • Typora(markdown编辑器)

      • 创新地将预览与写作界面合二为一,输入即输出,这个特性让它卓尔不群

      • 支持各大主流平台(Linux,Mac,Windows)

      • 拓展语法丰富,支持图表和公式的编写

      • 支持拖拽插入图片,这个功能不能少啊

      • 目前处于测试阶段(功能已经相当完善了),在测试期间完全免费的

    域名解析

    • 注册个人域名

      • 现在你的个人网站可以通过yourName.github.io进行访问,但很显然这个域名逼格不够,这就需要购买个人域名了,购买国内的域名需要经过繁琐的备案制度和严格的实名认证,备案需要一定的时间,但不会太长.

      • 常见的顶级域名如.com,edu,cn等,说实话真的不好注册,因为你选的一级域名有非常大的概率已经被注册,最终我只能在阿里云上选择注册一个比较冷门的顶级网域,正如你们所看到的:yanzehao.top,但好在价格十分便宜,一年只需要15人民币.

    • 添加域名解析

      • 最近一段时间发现在我的博客仓库的设置中GitHub Pages选项一直有这个警告:

        Your site’s DNS settings are using a custom subdomain, www.yanzehao.top, that’s set up as an A record. We recommend you change this to a CNAME record pointing at [YOUR USERNAME].github.io. For more information, see https://help.github.com/articles/setting-up-a-custom-domain-with-github-pages/.

        谷歌翻译了一下,大意就是:

        您网站的DNS设置正在使用自定义子域名www.yanzehao.top，该子域名设置为A记录。我们建议您将其更改为指向[您的用户名] .github.io的CNAME记录

      • 警告信息我一般都是选择性忽略,毕竟这并不影响我平常访问,可终归碍眼,那就按照GitHub的警告添加一条CNAME记录,随后GitHub的警告就彻底消失了

      • 为什么GitHub建议我使用CNAME记录来替代A记录?

        • A记录主要用来指定主机或者域名对应的IP地址,告诉DNS网站所在服务器的址，这样用户输入域名后DNS才能找到网站.
        • 不过有时候,我们的服务器的地址可能会随时发生变动,并不是固定不变的,这时候就要用到CNAME记录.cname又叫别名解析,一般是会指定一个DNS服务商提供的二级域名,这样DNS通过这个二级域名也能找到服务器的IP地址.
        • 使用CNAME的好处是,即使服务器的IP地址发生变更,也不用更改解析记录,DNS会自动更新解析,相比A记录少了一些改动的麻烦.

      • 在仓库设置中的GitHub Pages选项里勾选Enforce HTTPS单选框,从而使用更加安全的HTTPS协议.不知道GitHub从什么时候开始,在GitHub Pages中的自定义域名也支持HTTPS协议了,喜大普奔啊!!!为什么要使用HTTPS?

        HTTP协议传输的数据都是未加密的，也就是明文的，因此使用HTTP协议传输隐私信息非常不安全，为了保证这些隐私数据能加密传输，于是网景公司设计了SSL（Secure Sockets Layer）协议用于对HTTP协议传输的数据进行加密，从而就诞生了HTTPS。简单来说，HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，要比http协议安全。

        • HTTPS优点

          1. 使用HTTPS协议可认证用户和服务器，确保数据发送到正确的客户机和服务器
          2. HTTPS协议是由SSL+HTTP协议构建的可进行加密传输、身份认证的网络协议，要比http协议安全，可防止数据在传输过程中不被窃取、改变，确保数据的完整性
          3. HTTPS是现行架构下最安全的解决方案，虽然不是绝对安全，但它大幅增加了中间人攻击的成本。
          4. 谷歌曾在2014年8月份调整搜索引擎算法，并称“比起同等HTTP网站，采用HTTPS加密的网站在搜索结果中的排名将会更高”。
          5. 最重要的还是浏览器使用体验,之前我的blog使用HTTP协议时,chrome和Firefox每次都会醒目地提醒我此网站不安全.

        • HTTPS缺点

          1. HTTPS协议握手阶段比较费时,会使页面的加载时间延长近50%,增加10%到20%的耗电.
          2. HTTPS连接缓存不如HTTP高效,会增加数据开销和功耗,甚至已有的安全措施也会因此而受到影响.
          3. SSL证书需要钱,功能越强大的证书费用越高,个人网站,小网站没有必要一般不会用.
          4. SSL证书通常需要绑定IP,不能在同一IP上绑定多个域名,IPv4资源不可能支撑这个消耗.
          5. HTTPS协议的加密范围也比较有限,在黑客攻击,拒绝服务攻击,服务器劫持等方面几乎起不到什么作用.最关键的,SSL证书的信用链体系并不安全.特别是在某些国家可以控制CA根证书的情况下,中间人攻击一样可行.
