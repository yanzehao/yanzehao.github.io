介绍

web浏览器可能是使用最广泛的软件,当我们在浏览器地址栏输入www.baidu.com时开始,到我们看到百度主页为止,其中到底发生了什么?

从耗时的角度,浏览器请求,加载,渲染一个页面,时间耗费在下面五件事情上:

  1. DNS查询:域名解析,寻找服务器

  2. TCP连接:请求>确认>连接

  3. 客户端请求:

  4. 服务器应答:

  5. 客户端渲染

渲染引擎

由于技术有限,以上步骤涉及大量后端知识,本文只讨论第五个部分,即浏览器对接收到的数据的渲染,其中主要包括HTML文档,CSS样式表,JS脚本.主要由浏览器的渲染引擎完成.

  1. 解析数据

    • 浏览器将从服务器获取的HTML文档构建成文档对象模型DOM(Document Object Model).

  • 载入CSS样式表并解析，构成层叠样式表模型CSSOM(CSS Object Model).

  • JavaScript脚本文件被载入和解析,主要是通过DOM API和CSSOM API来操作DOM和CSSOM,

  1. 创建渲染树

    • 结合DOM和CSSOM，渲染树将会被创建，每一个渲染对象都包含了它相应的DOM对象以及计算后的样式.

    • 渲染树对应于DOM元素，但关系不是一对一的。非可视DOM元素不会插入渲染树中。一个例子是“head”元素。此外，display属性值为none的元素也不会出现在树中

    • 通过定位和盒子模型来精确计算每个元素的位置,称之为layout(布局)

  1. 渲染

    最后通过调用操作系统Native GUI的API进行渲染,称之为painting

以我土木转行的经验,这让我很容易想到cad制图中的layout(视口布局),这是较难理解的概念,在web中是完全相通的.

流程了解清楚后,可以稍作总结,这里使用树木进行类比,DOM相对于枝干;CSSOM相对于树叶;两者之间是通过选择器来形成结点,最后渲染树起到了园丁的作用,对每个元素的位置进行调整.

阻塞渲染:CSS与JavaScript

  1. 何为阻塞:

    HTML文档按照顺序结构自上而下逐行执行,加载CSS或JS这些文件影响了DOM的创建

  2. 具体原因

    • JavaScript的加载、解析与执行会阻塞DOM的构建
    • JavaScript 不仅可以读取和修改 DOM 属性，还可以读取和修改 CSSOM 属性,从而导致CSSOM也阻塞DOM的构建

回流与重绘

  • 重绘:当页面元素样式的改变不影响元素在文档流中的布局时，比如background-color.浏览器只会将新样式赋予元素并进行重绘操作
  • 回流:当页面元素样式的改变影响元素在文档流中的布局时，回流操作就会被触发

显而易见,回流所需的成本比重绘高得多,从性能和用户体验的角度讲,我们需要尽量避免回流
