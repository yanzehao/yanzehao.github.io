    需求

    • hexo生成的博客文章默认是按照文章生成的时间来进行排序的,最新生成的文章排在最前,这样的排序方式适合大多数场景.

    文章加密

    需求

    在个人博客的搭建及维护过程中,难免会涉及到一些个人隐私,如果不希望公开这些信息,就需要对文章进行加密

    插件

    hexo插件库里有现成的解决方案:hexo-blog-encrypt

    安装

    配置

    • 在站点配置文件中开启此插件的文章加密功能

    • 在文章源文件中配置密码,摘要,提示信息

    备注

    • 由于静态资源的局限性,这样的加密方式安全性较低,防君子不防小人,敏感信息不建议上传至博客

    静态资源压缩

    需求

    • 生成的博客代码中存在大量的缩进,注释,换行和空白,占用了部分空间,对文件进行压缩可减少客户端请求的数据量,提高站点访问速度
    • 压缩后的代码可读性变差,有利于提高站点安全性

    方案

    网上存在两种主流的文件压缩方案:

    • 自动化构建工具:gulp (功能强大,自定义程度高)
    • hexo插件:hexo-neat (快速便捷,推荐懒人或小白用户使用)

    域名分流解析

    前言

    GitHub生态的完整性是国内的git服务商无法比拟的,作为面向国际的开发者,无论是出于学习还是恰饭,GitHub毫无疑问是首选,但由于网络的限制,在中国大陆体验的并不是十分理想,特别是针对部署在GitHub pages上的静态博客而言,访问速度将大打折扣.对此,就需要对域名进行分流解析

    特性

    • 国内网络通过coding pages访问页面,国外网络通过GitHub pages访问
    • 有利于Google和百度对本站点进行收录

    方法

    1. 注册coding并创建一个项目来存放博客代码,基本上和GitHub一样,在coding上创建的这个项目的地址需要填写为yourName.coding.me,其他随意.

    2. 在coding pages服务中绑定自定义域名,绑定前请在域名 DNS 设置中添加一条CNAME记录指向nullboy.coding.me,关键点在于解析路线的选择,我的设置如下图.

      • 绑定的域名需要与DNS解析中的主机记录与记录值对应,例如主机记录为www,那么你的自定义域名也必须加上www
      • 一般是强制https访问,同时需要在coding pages上申请免费的安全证书,在申请安全证书之前,需要暂停GitHub pages的DNS解析,申请成功之后再将其开启

    3. 将代码同时推送到GitHub与coding上对应的仓库

      • 同GitHub一样,需要添加coding账户的SSH公钥

      • 在站点配置文件_config.yml中添加相应的配置,随后即可进行部署

    4. 同一资源文件在不同网络环境下向不同的服务器请求资源,至此即完成了分流解析

      • 国外网络通过GitHub进行访问

      • 国内网络通过coding进行访问
